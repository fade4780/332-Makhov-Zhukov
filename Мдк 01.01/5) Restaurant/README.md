# Система управления ресторанами

Консольное приложение для управления сетью ресторанов с поддержкой меню, заказов и акций.

## Возможности

- **Управление ресторанами**: 3 филиала в Москве, 2 в Санкт-Петербурге
- **Меню**: Блюда и напитки с различными типами и характеристиками
- **Динамическое ценообразование**: Цены в Москве на 10% выше, чем в Санкт-Петербурге
- **Временные меню**:
  - Бизнес-ланчи доступны с 12:00 до 15:00
  - Утреннее меню доступно с 7:00 до 11:00
- **Акции**: Скидка 20% на еду с 15:00 до 18:00 по будням
- **Блюда от шефа**: Уникальные блюда для каждого ресторана
- **Изменение цен**: Возможность изменения цен каждой позиции
- **Заказы**: Создание заказов с автоматическим расчетом итоговой суммы

## Технологии

- Java 11
- SQLite (база данных)
- Maven (система сборки)
- JDBC (работа с БД)
- Лямбда-выражения для фильтрации и обработки данных

## Структура проекта

```
src/main/java/com/restaurant/
├── model/          # Модели данных (Dish, Drink, Restaurant, Order и т.д.)
├── dao/            # Data Access Object для работы с БД
├── service/         # Бизнес-логика (MenuService, OrderService, PromotionService)
├── ui/              # Консольный интерфейс
└── util/            # Утилиты (инициализация БД)
```

## Установка и запуск

### Требования

- Java 11 или выше
- Maven 3.6+

### Сборка проекта

```bash
mvn clean compile
```

### Запуск приложения

```bash
mvn exec:java -Dexec.mainClass="com.restaurant.Main"
```

Или после сборки:

```bash
mvn package
java -jar target/restaurant-management-1.0-SNAPSHOT.jar
```

## Использование

1. При запуске приложения база данных автоматически инициализируется с тестовыми данными
2. Выберите ресторан из списка
3. Просмотрите меню (меню зависит от текущего времени)
4. Добавьте блюда и напитки в заказ
5. Завершите заказ - будет автоматически рассчитан итоговый чек с учетом акций

## Особенности реализации

- **ООП принципы**: Использование инкапсуляции, наследования через enum, полиморфизма
- **Лямбда-выражения**: Использованы для фильтрации меню, обработки заказов и вывода данных
- **База данных**: SQLite для хранения данных о ресторанах, блюдах, напитках и заказах
- **Отсутствие статики**: Все методы и поля являются экземплярными, что соответствует принципам ООП

## Примеры использования

### Изменение времени

В главном меню выберите пункт "3. Установить время" для тестирования различных временных меню и акций.

### Изменение цен

В главном меню выберите пункт "2. Изменить цены" для изменения базовых цен блюд и напитков.

## База данных

База данных SQLite создается автоматически при первом запуске в файле `restaurant.db` в корне проекта.

Таблицы:
- `dishes` - блюда
- `drinks` - напитки
- `restaurants` - рестораны
- `orders` - заказы
- `order_items` - позиции заказов
